# Contract coverage report using SOAP UI

<p><span style="font-family: Constantia,serif"><span style="font-size: small">My previous project was with a Java Web Service Provider team. Being the service provider, it was very critical for us to keep the web services up and running. A little outage or bugs impact many groups' work. Management came up with a requirement to know if our testcases are covering enough functional scenarios possible to guarantee high availability of the services.</span></span></p>

<p><p><span style="font-family: Constantia,serif"><span style="font-size: small">We were using SOAP UI for unit testing web services. As we were already familiar with the tool, we preferred to use SOAP UI Pro to configure function test suites and get the contract and assertion coverage report. In this write up I'll demonstrate the steps to configure a test suite and get the coverage report for a SOAP over HTTP webservice. <!--more--></span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: medium"><b>Step -1 Create a SOAP UI project:</b></span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small">Go to the file menu -&gt; New soapUI Project. Give the project any name and select your WSDL file. Click on “OK” and you'll see your test project in left area.</span></span></p> <p><a href="http://xebee.xebia.in/wp-content/uploads/2013/03/pic1.png"><img class="aligncenter size-full wp-image-16280" alt="pic1" src="http://xebee.xebia.in/wp-content/uploads/2013/03/pic1.png" width="680" height="386" /></a></p> <p><span style="font-family: Constantia,serif"><span style="font-size: medium"><b>Step -2 Create a test suite</b><span style="font-family: Consolas,monospace"><b>:</b></span></span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small">Open the project created in step 1 and right click on web service name. In the right click menu, click on 'Generate Test Suite' option.</span></span></p> <p><a href="http://xebee.xebia.in/wp-content/uploads/2013/03/pic21.png"><img class="aligncenter size-full wp-image-16318" alt="pic2" src="http://xebee.xebia.in/wp-content/uploads/2013/03/pic21.png" width="664" height="325" /></a></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small"> In this dialog there will be two options to select. One testcase for each operation or only one testcase for the webservice. I prefer first style. By using this I can see coverage for each operation clearly. Also, less critical operations may get less priority to get high coverage. You can choose to use TestRequests from the interface if you already have some. Otherwise you can choose to start with empty requests. Click on 'OK', choose a name for test suite and go ahead.</span></span></p> <p><a href="http://xebee.xebia.in/wp-content/uploads/2013/03/pic3.png"><img class="aligncenter size-full wp-image-16282" alt="pic3" src="http://xebee.xebia.in/wp-content/uploads/2013/03/pic3.png" width="655" height="298" /></a></p> <p><span style="font-family: Constantia,serif"><span style="font-size: medium"><b>Step -3 Adding Test Steps to the test suite<span style="font-family: Consolas,monospace">:</span></b></span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small">One test step will be already added to the testcase with empty tetrequest. To add more, select the testcase and right click. In the menu 'Add Step' -&gt; 'Test Request'. Give it a name. Select the operation you want to invoke for this step. (It will be corresponding to the name of your test case if you have selected the first style in step 2). Add request to testcase. Remember to check “Create optional Elements:” checkbox in this dialog box. Otherwise you'll get the complete request skeleton to fill with data.</span></span></p> <p><a href="http://xebee.xebia.in/wp-content/uploads/2013/03/pic4.png"><img class="aligncenter size-full wp-image-16283" alt="pic4" src="http://xebee.xebia.in/wp-content/uploads/2013/03/pic4.png" width="647" height="335" /></a></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small">Populate the request with your functional test scenario. Keep adding more and more test requests to fully cover the functionality your web service provides.</span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: medium"><b>Step -4 Setting endpoint and credential information:</b></span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small">Now, the test suite is ready. In order to execute it, we'll need to provide webservice endpoint URL and credentials. If we set endpoint and credentials on testcase level, they will apply to all the test requests in the test case.</span></span></p> <p><a href="http://xebee.xebia.in/wp-content/uploads/2013/03/pic5.png"><img class="aligncenter size-full wp-image-16284" alt="pic5" src="http://xebee.xebia.in/wp-content/uploads/2013/03/pic5.png" width="671" height="73" /></a></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small">Click on the 'key' to set the credentials and 'URL' to set endpoint information.</span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: medium"><b>Step -5 Executing test suite:</b></span></span></p> <p><span style="font-family: Constantia,serif"><span style="font-size: small">Before running the test suite, we'll enable the coverage option so that we can generate coverage report later. Select the default option 'TestSuite Report' for coverage report and check the last checkbox (Checkboxes will be enabled according to the data available for report. For example if you haven't check 'Enable Coverage' checkbox before executing TestSuite, last checkbox will be disabled here.). Click on 'OK' and coverage results will be exported into report. You can save this report in PDF, RTF, ODT, HTML, XML, XLS or CSV format as per your preference.</span></span></p></p>
